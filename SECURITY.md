# Security Policy

## Supported Versions

Şu anda desteklenen versiyonlar:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

Güvenlik açığı bulduysanız, lütfen aşağıdaki adımları izleyin:

1. **Public issue açmayın** - Güvenlik açıkları public olarak paylaşılmamalıdır
2. Email gönderin: bulusluemircan723@gmail.com
3. Aşağıdaki bilgileri ekleyin:
   - Açığın detaylı açıklaması
   - Nasıl tekrarlanacağı
   - Potansiyel etkisi
   - Varsa önerilen çözüm

## Response Time

- İlk yanıt: 48 saat içinde
- Düzeltme: Kritik açıklar için 7 gün, diğerleri için 30 gün

## Disclosure Policy

Güvenlik açığı düzeltildikten sonra:
1. Yeni versiyon yayınlanır
2. CHANGELOG'da belirtilir
3. GitHub Security Advisory oluşturulur
4. Keşfeden kişiye teşekkür edilir (istenirse)

## Security Best Practices

Dev:Health kullanırken:
- ✅ Sadece development ortamında kullanın
- ✅ Production'da `--dev` flag ile yükleyin
- ✅ Raporları güvenli yerlerde saklayın
- ✅ Hassas bilgileri içeren raporları paylaşmayın

Teşekkürler! 🙏
